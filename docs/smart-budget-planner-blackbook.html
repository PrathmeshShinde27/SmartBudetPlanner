1. Introduction

Smart Budget Planner is a web application that helps a user plan monthly income, create category-wise budgets, record expenses, and understand where money is being spent.

What the Project Is About

This project converts the idea of an Excel budget tracker into a proper website. In Excel, a user can enter planned budgets and daily spending manually. In this web app, the same logic is made easier through forms, dashboards, charts, filters, authentication, and a database.

Problem Statement

Many people track money in notebooks or Excel files. This works for simple use, but it becomes difficult when the user wants login security, mobile access, monthly history, automatic totals, charts, and safe storage. The problem is to build a digital system where each user can securely manage personal budget data.

Goal of the Project

The goal is to build a production-ready budget planner that can run locally during development and later go live on a real domain using free hosting services.

2. Project Overview

The application has four main parts: a user interface, a backend API, a PostgreSQL database, and an email service for OTP verification.

What the Application Does

  • Allows new users to create an account.
  • Verifies email using OTP before login.
  • Lets users enter monthly income.
  • Lets users create categories like Food, Travel, Bills, Savings, or NPF Fund.
  • Stores planned budget and actual expenses.
  • Shows dashboard cards, pie chart, bar chart, and category comparison.
  • Supports edit, delete, monthly filtering, and Excel export.

Key Features

Email + Password Login Email OTP User Data Isolation Monthly Budget Expense CRUD Charts Admin View Excel Export Mobile Responsive

High-Level Explanation

When a user opens the website, React displays the screens. When the user logs in or adds an expense, React sends a request to the Express backend. The backend checks security, validates data, talks to PostgreSQL, and returns the result. The frontend then updates the page without reloading.

How the Parts Connect

  1. The browser loads the React frontend.
  2. React uses Axios to call backend API endpoints.
  3. The backend checks JWT token for protected routes.
  4. The backend reads or writes data in PostgreSQL.
  5. The result is sent back as JSON.
  6. React updates the dashboard instantly.

5. Modules Explanation

🔐 Authentication Module

What it does: Handles register, login, email verification, forgot password, reset password, profile update, and password update.

How it works: During signup, the password is hashed with bcrypt and saved. An OTP is emailed to the user. After OTP verification, the user can log in and receives a JWT token.

🔌 API Module

What it does: Provides REST endpoints such as /dashboard, /categories, /expense, and /income.

How it works: Each route receives a request, validates input, checks authentication, performs database queries, and returns JSON.

🐘 Database Module

What it does: Stores user accounts, OTP records, income, categories, monthly budgets, and expenses.

How it works: Tables are connected with foreign keys. Every financial table stores a user_id, so one user's data does not mix with another user's data.

🎨 Frontend Module

What it does: Shows login pages, dashboard, expense page, category page, settings page, and admin page.

How it works: React components store form values in state. After API success, the component refreshes data and shows a top-center toast message.

📧 Email OTP System

What it does: Sends one-time passwords for email verification and password reset.

How it works: The server creates a short OTP, stores only its hash, sends the OTP through Resend, and verifies it before marking email verified or resetting password.

7. Database

The project uses PostgreSQL, hosted on Neon in production. PostgreSQL is a strong relational database, which is suitable for financial records because relationships and data rules are important.

Why PostgreSQL Was Chosen

  • It stores data in structured tables.
  • It supports relations using foreign keys.
  • It can calculate totals using SQL aggregation.
  • It is reliable for money-related records.
  • Neon provides a free hosted PostgreSQL option.

8. Authentication & Security

Login and Signup System

Users sign up with name, email, phone country code, phone number, city, and password. After signup, the user must verify email with OTP. Login is allowed only after email verification.

Email OTP Verification

  1. User enters email during signup or forgot password.
  2. Backend generates an OTP.
  3. Backend stores only a hashed version of the OTP.
  4. Resend sends the OTP email from the verified domain email.
  5. User enters OTP in the app.
  6. Backend checks the OTP and marks it as consumed.

Password Security

Passwords are never stored directly. The backend uses bcrypt to convert the password into a hash. A hash is like a one-way locked version of the password. Even if someone sees the database, they cannot read the original password.

JWT Token

After login, the server gives a signed JWT token. The frontend sends this token with protected requests. The backend checks the token to know which user is making the request.

Environment Variables

Secrets like database URL, JWT secret, and Resend API key are stored in environment variables. They are not written in the source code.

Important: Real passwords, real API keys, and real database URLs should never be included in a report, GitHub repository, or screenshot.

9. Docker & Deployment

What Docker Is

Docker is a tool that packages software into containers. A container is like a small ready-made environment that includes everything needed to run a service. For example, PostgreSQL can run inside a Docker container without installing PostgreSQL directly on the computer.

How Docker Was Used

Docker was used for local PostgreSQL testing through docker-compose.yml. This lets the developer test with a real PostgreSQL database before using the hosted Neon database.

docker compose up -d postgres

10. Domain & Hosting

Domain Purchased via GoDaddy

The project domain is prathmeshshinde.com. A domain is the human-readable website name. Instead of remembering a technical hosting URL, users can open the project through a custom domain or subdomain.

How the Domain Is Connected

  1. Buy or manage the domain in GoDaddy.
  2. Create frontend project in Vercel.
  3. Add the custom domain or subdomain in Vercel project settings.
  4. Copy DNS records shown by Vercel.
  5. Paste those records in GoDaddy DNS management.
  6. Wait for DNS propagation.

Important Budget Logic

  • Total Budget: Monthly income/budget entered by user.
  • Total Planned: Sum of all category planned budgets.
  • Total Spent: Sum of all expenses for selected month.
  • Remaining: Total Budget minus Total Spent.
  • Category Actual: Sum of expenses for that category in selected month.
  • Warning: Show alert only when spending crosses planned category budget.

14. Conclusion

What Was Achieved

Smart Budget Planner was successfully built as a full-stack web application. It supports real user accounts, email OTP verification, PostgreSQL database storage, monthly budgeting, category planning, daily expense logging, dashboard charts, Excel export, and deployment-ready configuration.

What Was Learned

  • How to convert Excel logic into a real web application.
  • How frontend, backend, database, and APIs connect.
  • How to store financial data securely per user.
  • How password hashing and JWT authentication work.
  • How email OTP improves account verification.
  • How to deploy a project using Vercel, Render, Neon, and a GoDaddy domain.

Future Improvements

  • Add stronger admin controls with audit logs.
  • Add recurring expenses like rent or subscriptions.
  • Add email alerts when a category crosses planned budget.
  • Add PDF reports for monthly summary.
  • Add bank statement import with automatic categorization.
  • Add two-factor authentication for admin users.
Final Result: The project is simple enough for daily use, but technically strong enough to be hosted as a real website with separate frontend, backend, and database services.
